# Cursor Hackathon: Build What Eric Couldn't

## Event Overview

Welcome to the Cursor Hackathon at Dawn Capital HQ! This event challenges participants to build implementations of features and ideas that Eric from Cursor has posted about but hasn't yet shipped. Your mission: prove you can build it better, faster, and more creatively than the Cursor team.

## Event Details

- **Venue:** Dawn Capital HQ (Terrace and interior spaces available)
- **Date:** Early March 2026 (TBD based on venue availability)
- **Capacity:** 100 participants
- **Event Type:** Competitive Hackathon with AI/Coding Tools Focus
- **Duration:** Full day event (specific hours TBD)

## Theme & Concept

**"Do What Eric Couldn't"** - This hackathon is built around a curated "hit list" of Eric Zakariasson' open-ended X/Twitter posts about Cursor features, improvements, and ideas that haven't been implemented yet. Each post represents a challenge for participants to tackle.

The goal is to inspire developers and demonstrate what's possible when the community takes on features that even leading AI coding tools are still working on. This is your chance to push boundaries and show what can be built with creativity and determination.

## Challenge Structure

All challenges are sourced from an Airtable database containing Eric' posts. Each challenge includes:

- Original post/tweet content
- Feature description or idea
- Suggested implementation approach (optional)
- Difficulty level estimate
- Judging criteria specific to that challenge

### The "Hit List" (Hackathon Tracks)

Based on Eric Zakariasson's recent posts and ideas, participants can choose from the following tracks:

#### 1. The "LLM Phrase Collection" / Debugging Analytics Track
*   **The Post:** "we should start a collection of the best LLM phrases. i’ll go first: Ah, I see the issue now"
*   **The Challenge:** Build a tool that automatically detects, collects, and categorizes common LLM phrases, "hallucination tells", or debugging loops during agentic workflows. Use this data to help developers understand when their AI is stuck in a loop or starting to hallucinate.

#### 2. The "Subagent Orchestration" Track
*   **The Idea:** Inspired by the concept of specialized AI agents collaborating on complex tasks, and the need for robust "Computer Use" capabilities.
*   **The Challenge:** Design and implement a system where a primary AI agent can dynamically spawn, manage, and coordinate multiple specialized "subagents" to tackle a larger coding project. This could involve subagents for testing, documentation, UI design, or specific backend tasks, all working in concert within a simulated or real OS environment.

#### 3. The "Vibe Arena" / Collaborative AI Development Track
*   **The Idea:** Envisioning a future where AI and human developers collaborate seamlessly in shared, dynamic environments.
*   **The Challenge:** Create a "Vibe Arena" – a collaborative coding space that intelligently adapts to the "vibe" or state of the development team (human and AI). This could involve dynamic resource allocation, context-aware communication tools, or AI-driven suggestions that anticipate needs based on team activity and project status.

#### 4. The "Just-in-Time Tools" / Contextual Productivity Track
*   **The Idea:** Building highly intelligent, context-aware tools that appear precisely when needed, minimizing distractions and maximizing flow.
*   **The Challenge:** Develop a suite of "Just-in-Time" developer tools that integrate deeply into the coding environment. These tools should proactively offer assistance, suggest refactorings, provide documentation, or even generate code snippets based on the developer's current context, without explicit prompting, and then recede when no longer relevant.

#### 5. The "Long-Running Agents" / Autonomous Project Management Track
*   **The Idea:** Inspired by the launch of **GPT-5.3 Codex** (Feb 09, 2026) and the potential for agents to manage complex, multi-day projects (1,000 commits per hour!).
*   **The Challenge:** Build a "Long-Running Agent" system using **Opus 4.6 (Fast Mode)** for architecture and Codex for execution. It must handle multi-day project management, error recovery, and autonomous PR creation without human intervention.

#### 6. The "Video-is-Code" / Visual QA Track
*   **The Idea:** Inspired by Eric's posts "video is code" and "Ralph Wiggum with visual QA as verifier."
*   **The Challenge:** Build a tool that uses Cursor's **Computer Use** to perform visual regression testing. It should record the UI, detect "vibe" discrepancies or layout shifts, and automatically generate the CSS/React fixes.

## Judging Criteria

- **Implementation Quality:** Does it work? Is the code clean and well-structured?
- **Feature Completeness:** How fully does it address the challenge from Eric' post?
- **Innovation:** Did the team find creative approaches or improvements beyond the original idea?
- **User Experience:** Is it intuitive and useful for developers?
- **Technical Difficulty:** Complexity of the challenge tackled
- **Presentation:** Quality of demo and explanation
